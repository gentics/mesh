Constructs

Summary

Here we describe a common internal structure of a construct (aka tag type).

Normally, a construct consists of a gentics_structure.json file, containing the basic construct features, and zero or more part.<part_keyword>.<part_type_extension> files with the values for an each construct part, enclosed in the same folder.

File structure

Construct's gentics_structure file

The gentics_structure.json file is a JSON file with the following fields:

FieldDescriptionMandatoryAllowed valuesDefault valueExample 
globalIdA globally unique, automatically generated identifier.YesMust follow the Regexp format: [0-9a-z]{4}\\.[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12} 
keywordA construct identifier, unique across all constructs.YesAlphanumerical value, max 255 characters long 
nameA set of human readable construct names.Yeskey:value pair, where key corresponds to the language ISO 639-1:2002 code, and value is an arbitrary string in this languageAt least one pair has to be defined { "en": "Semver", "de": "Semantische Version" }
descriptionA set of human readable construct descriptions.Nokey:value pair, where key corresponds to the language ISO 639-1:2002 code, and value{ "en": "A description of a version in a format of `major.minor.patch` numbers", "de": "Eine Beschreibung der Version laut Nummern mit Formatierung `major.minor.patch`" } 
mayBeSubtagA flag, marking this construct as allowed to be inserted into other constructs.Notrue or falsefalse 
mayContainsSubtagsA flag, marking this construct as allowed to contain other constructs. Notrue or falsefalse 
editOnInsertA flag, marking the instance of this construct to open its editor upon its initialization in a page.Notrue or falsefalse 
editorControlStyleA definition of how the controls, that belong to this construct, should be drawn in the edit mode, for an optimal edit mode UX. No- ABOVE (controls are above the construct instance, default value)
- ASIDE (controls are placed left to the construct instance), 
- CLICK (no controls, shown, the tag editor is opened with a click on the control instance)		ABOVE

autoEnableA flag, marking the instance of this construct to be automatically enabled upon its initialization in a page. Notrue or falsefalse 
externalEditorUrlA string value of an URL of an external editor for the instances of this construct. NoA valid URL 
partsAn array of zero or more construct part definitions (see below).YesAn array of construct part definitions

A JSON array, may be empty

 

Part definition structure

Each of the definition of a construct part consists of a definition in the parts array of a gentics_structure.json file, and a default value, contained in an own part.<part_keyword>.<part_type_extension> file. The JSON structure of a part definition looks like following:

FieldDescriptionMandatoryAllowed valuesDefault value Example
globalIdA globally unique, automatically generated identifier.YesMust follow the Regexp format: [0-9a-z]{4}\\.[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12} 
keywordA construct identifier, unique across all constructs.YesAlphanumerical value, max 255 characters long 
nameA set of human readable construct names.Yeskey:value pair, where key corresponds to the language ISO 639-1:2002 code, and value is an arbitrary string in this languageAt least one pair has to be defined{ "en": "Major version", "de": "Majorversion" } 
typeIdAn ID of the type of this construct part.YesAn integer. See list of type IDs and their descriptions. 
orderA value of a part's order within the parts array, unique for this construct.YesAn integer. Typically the count numbers are chosen as a value. 
mlIdAn ID of a markup language, used in this part.Yes0

Each construct part has a part.<part_keyword>.<part_type_extension> file, which contains a default value for the part. For instance, if a part is named semver, and has a type Text (typeId == 1), its value file will be named part.semver.txt. If there is no default value required for semver, this file still has to exist, though remain empty. (TODO: describe all the possible types and extensions).
